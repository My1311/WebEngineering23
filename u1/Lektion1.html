1.1 World Wide Web

1. Mit welchen fachlichen Argumenten wurde das WWW-Proposal von Tim Burners Lee abgelehnt?

Die fachlichen Argumenten, warum das WWW-Proposal von Tim Burners Lee abgelehnt wurde, sind:

  • fehlende mathematische Rigorosität,
  • Gefahr von “broken links”,
  • Verteilte Texte statt verteilte Objekte,
  • besser wäre ein Objekt- statt einem Text-Protokoll,
  • “Texte ohne klar definierte Semantik sind die Ursache allen Übels.”,
  • Ohne klar definierte Semantik ergibt sich eine lange Liste von Patches.

    • 2. Was sind die fachlichen Argumente, warum das WWW dennoch ein Erfolg wurde?

    Die fachlichen Argumente, warum das WWW dennoch ein Erfolg wurde, sind:

  • Technologie von WWW ist simpel. => fast verbreitet,
  • WWW ist ein Mittel, sodass alle Rechner in der Welt miteinander kommunizieren können.

    • 3. Was wäre der Preis für die garantierte Verhinderung von "broken links"?

    Der Preis für die garantierte Verhinderung von "broken links" wäre:

  • Vermeidung der Deep-Links auf der Website, es sei denn (oder bis) dies erforderlich ist,
  • Einsatz von Umleitungsmechanismen,
  • Erstellung und Ersatz des Inhaltes der defekten URL neu,
  • Leitung der defekten Seite (301) auf eine andere relevante Seite Ihrer Website um,
  • 404: Dies ist eine Option; Sie müssen nur sicherstellen, dass Sie einen „harten“ 404 und keinen „weichen“ 404 anzeigen.

    • 1.2 HTTP

    1. Sie bekommen im Browser den HTTP Status Code 200. Was bedeutet das?

    2. Sie bekommen im Browser den HTTP Status Code 301. Was hat das zu bedeuten?

    3. Sie bekommen im Browser den HTTP Status Code 400. Was hat das zu bedeuten? Was können Sie dagegen tun?

    4. Sie bekommen im Browser den HTTP Status Code 403. Was hat das zu bedeuten? Was können Sie dagegen tun?

    5. In einer Webanwendung benötigen Sie eine OPTIONS-Anfrage, die die Optionen des Servers vor dem eigentlichen Zugriff erfragen soll. Aus Performancegründen soll die Abfrage aber cacheable sein. Wie könnten Sie dafür eine Lösung angehen?

    1. Der HTTP-Statuscode 200 OK gibt an, dass eine Anfrage erfolgreich verlaufen ist. Eine 200-Antwort ist standardmäßig im Cache speicherbar. Unter dieser URL wurde eine valide Webseite gefunden.

    2. Code 301: Die URL wurde dauerhaft auf eine neue URL verschoben.

    3. Code 400: Der HTTP-Statuscode 400 Bad Request gibt an, dass der Server die Anfrage nicht verarbeiten kann, weil anscheinend ein clientseitiger Fehler geschehen ist (z.B. eine syntaktisch falsche Anfrage). Der Client sollte diese Anfrage nicht unmodifiziert wiederholen.

    4. Code 403: Der HTTP 403 Forbidden Statuscode zeigt an, dass der Server die Anfrage zwar verstanden hat, aber sich weigert, sie zuzulassen.

    5. OPTIONS-Anfrage ist nicht cacheable.

    1.6 HTML-Literatur lesen und Fragen beantworten

    1. Was ist HTML?

    HTML (Hypertext Markup Language) ist der Code, welcher benötigt wird, um den Webinhalt zu strukturieren und ihm eine Bedeutung und einen Zweck zu geben.

    2. Wie kann man eine geschachtelte geordnete Liste der Tiefe 3 erzeugen?

    Mit <ul>, <li>, <ol> wie zB: <html> <head> <title>verschachtelte Listen</title> </head> <body> <ul> <li>Erster Listeneintrag</li> <li>Zweiter Listeneintrag</li> <ol> <li>Unterpunkt 1 des zweiten Listenpunktes</li> <li>Unterpunkt 2 des zweiten Listenpunktes</li> <li>Unterpunkt 3 des zweiten Listenpunktes</li> </ol> <li>Dritter Listeneintrag</li> <li>Vierter Listeneintrag</li> </ul> </body> </html>

    3. Wie ist eine HTML-Tabelle aufgebaut?

    Eine HTML-Tabelle wird wie zB aufgebaut:

    <table> <tr> <td>Inhalt</td> </tr> </table>

    Dieses (sinnfreie) Beispiel definiert eine Tabelle, welche nur aus einer einzigen Zelle besteht.

                            
<table> leitet eine Tabelle ein (table = Tabelle).
<tr> leitet eine neue Tabellenzeile ein (tr = table row = Tabellenzeile).
<th> leitet eine Kopfzelle ein (th = table header = Tabellenkopf),
<td> eine normale Datenzelle (td = table data = Tabellendaten).

    4. Welche Konventionen sollte man bei Pfaden und Dateinamen beachten?

    Bei modernen Unix-Systemen, wie die meisten WWW-Server eingesetzt wird, sind Dateinamen bis zu 256 Zeichen Länge erlaubt. In jedem Fall deutsche Umlaute und scharfes ß in den Dateinamen sollten vermieden werden. Als Sonderzeichen ist in jedem Fall der Unterstrich "_" erlaubt. Andere Satzzeichen sollten auch vermieden werden.. Fragezeichen "?", Sternzeichen "*", Istgleichzeichen "=" dürfen auf keinen Fall in Dateinamen verwendet werden. Auch auf Leerzeichen sollten unbedingt verzichtet werden.

    5. Wie baut man in HTML ein Menü?

    Eine Navigationsleiste ist im Grunde eine Liste von Links, daher ist die Verwendung der Elemente <ul> und <li> absolut sinnvoll: <!DOCTYPE html> <html> <body> <ul> <li><a href="#home">Startseite</a></li> <li><a href="#news">Neuigkeiten</a></li> <li><a href="#contact">Kontakt</a></li> <li><a href="#about">Über</a></li> </ul> <p>Hinweis: Wir verwenden href="#" für Testlinks. Auf einer echten Website wären dies URLs.</p> </body> </html>

    6. Welche Attribute sollte man bei Bildern wie verwenden?

    Das Attribut src (source) gibt den Namen oder die Position des anzuzeigenden Bildes an. Der Wert des Attributs src muss den Namen der Bilddatei oder deren URL enthalten. Das Attribut alt ist für das Image-Tag auch notwendig. Es wird verwendet, um Browsern einen alternativen Text zur Verfügung zu stellen, wenn das Bild nicht angezeigt werden kann (bei langsamer Verbindung, wenn ein Bildschirmleser verwendet wird usw.). Es wird empfohlen, die Attribute width und height mit dem Tag <img> zu verwenden. Die Atrribute class klassifiziert das Bild, sodass wir sie einfach mit CSS dekorieren können. zB: <img src="img_lights.jpg" class="w3-round-small" alt="Norway" style="width:30%">
