10.1 Static classes in TypeScript

Gibt es in TypeScript statische Klassen?

Erläutern Sie den Unterschied von TypeScript zu anderen objekt-orientierten, statisch typisierten Programmiersprachen wie Java und C#.

In TypeScript gibt es keine expliziten statischen Klassen wie in Sprachen wie Java oder C#.
In TypeScript sind Klassen selbst bereits statisch, wenn sie als solche deklariert werden.
Das bedeutet, dass die Methoden und Eigenschaften einer Klasse auf der Klasse selbst definiert sind und nicht auf Instanzen dieser Klasse.

Die Unterschiede von TypeScript zu Java und C# sind:

syntaktischen Unterscheidung zwischen statischen und nicht-statischen Elementen in diesen Sprachen. In TypeScript gibt es keine explizite Notwendigkeit für das Schlüsselwort static bei der Definition von Methoden oder Eigenschaften einer Klasse, da alle Klassenmitglieder per Definition statisch sind.

TypeScript ist eine von Microsoft entwickelte Programmiersprache, die auf JavaScript aufbaut und optional statische Typisierung unterstützt. Java und C# sind vollständig statisch typisierte Sprachen.

TypeScript hat mehr Flexibilität und Dynamik, da es mit dynamisch typisierten JavaScript-Bibliotheken und Frameworks integriert werden kann. Java und C# hingegen sind strenger in Bezug auf statische Typisierung und bieten eine striktere Kontrolle über den Typen während der Kompilierung. In TypeScript haben wir die Möglichkeit, Variablen aus zwei verschiedenen Klassen ,die gleiche strukturiert sind , in einer Funktion zu lesen und zu ändern

10.2 Fehler finden mit TypeScript

Gegeben sei folgendes fehlerbehaftete JavaScript-Programm.
Ergänzen Sie dieses um möglichst viele Typisierungen und übersetzen Sie es in ein TypeScript-Programm. 

                            
const add = (x,y) => x+y;
const equals = (x,y) => x===y;

console.log( add(1,2) );
console.log( add(true, true ) );
console.log( add(true, false ) );
var x = y = 1;
console.log( add(equals(x,y), equals(y,x)) );

Meine Lösung

TypeScript-Programm: const add = (x: number, y: number): number => x + y; const equals = <T>(x: T, y: T): boolean => x === y; console.log(add(1, 2)); // Ausgabe: 3 console.log(add(true, true)); // TypeScript gibt möglicherweise einen Fehler aus: Argument vom Typ 'boolean' ist nicht zuweisbar zu Parameter vom Typ 'number' console.log(add(true, false)); // TypeScript gibt möglicherweise einen Fehler aus: Argument vom Typ 'boolean' ist nicht zuweisbar zu Parameter vom Typ 'number' // Variablen mit Typisierungen let x: number = 1; let y: number = 1; // Hier ist eine mögliche Art, die Typisierung zu korrigieren // var x = y = 1; wird ersetzt durch: let x: number = 1; let y: number = 1; console.log(add(equals(x, y), equals(y, x))); // TypeScript gibt möglicherweise einen Fehler aus: Argument vom Typ 'boolean' ist nicht zuweisbar zu Parameter vom Typ 'number'

Was ist der Fehler im gegebenen JavaScript-Code? Welche Fehler meldet TypeScript?

Meine Lösung

Unzureichende Typisierung: const add = (x,y) => x+y; const equals = (x,y) => x===y; console.log( add(1,2) ); // Ausgabe: 3 console.log( add(true, true ) ); // Ausgabe: 2, weil true als 1 umgewandelt wird. console.log( add(true, false ) );//Ausgabe: 1, weil false als 0 umgewandelt wird. var x = y = 1; // y ist nicht definiert console.log( add(equals(x,y), equals(y,x)) ); // Fehler, aber wenn wir var x = 1; var y = 1 definieren, bekommen hier 2 Bei TypeScript erkennt man solche Fehler: // Fehler: Funktion 'add' erwartet number-Typen, aber erhält boolean-Typen console.log(add(true, true)); // Fehler: Argument vom Typ 'boolean' ist nicht zuweisbar zu Parameter vom Typ 'number' console.log(add(equals(x, y), equals(y, x))); // Möglicherweise unsichtbarer Fehler: y wird global initialisiert, x wird lokal initialisiert // In TypeScript würde man 'let' statt 'var' verwenden, um diesen Fehler zu vermeiden var x = y = 1;
